问题 2：make modules 非常慢

  • 使用多核编译：sudo make -j4 modules（4 为 CPU 核心数）
  • 虚拟机分配更多 CPU 和内存

问题 4：系统调用返回 -1

解决方案：

  • 检查系统调用号是否正确
  • 确认 NR_syscalls 已更新
  • 查看 dmesg 输出是否有错误信息

✅ 总结与建议

关键要点

  1. 耐心：内核编译耗时较长，尤其是 make modules
  2. 备份：修改前备份重要文件
  3. 版本匹配：确保内核版本与教程一致
  4. 日志查看：善用 dmesg 调试

学习收获

  • 理解 Linux 内核编译流程
